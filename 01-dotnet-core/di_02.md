# 🔥 1. How would you implement “Named Services” in .NET Core DI?

## ❓ Problem

.NET DI does NOT support named services like:

```
GetService("email")
```

---

## ✅ Solution (Factory + Dictionary Pattern)

```csharp
public interface INotificationService
{
    string Type { get; }
    Task SendAsync(string msg);
}
```

```csharp
public class EmailService : INotificationService
{
    public string Type => "email";
    public Task SendAsync(string msg) => Task.CompletedTask;
}
```

---

### Register

```csharp
services.AddScoped<INotificationService, EmailService>();
services.AddScoped<INotificationService, SmsService>();
```

---

### Resolver

```csharp
public class NotificationResolver
{
    private readonly Dictionary<string, INotificationService> _map;

    public NotificationResolver(IEnumerable<INotificationService> services)
    {
        _map = services.ToDictionary(x => x.Type, StringComparer.OrdinalIgnoreCase);
    }

    public INotificationService Resolve(string type)
    {
        return _map[type];
    }
}
```

---

## 🎯 Interview Insight

👉 Shows:

* Workaround for missing feature
* Clean extensibility
* Strategy pattern

# 🔥 2. How do you handle Circular Dependencies in DI?

## ❌ Problem

```csharp
class A { public A(B b) {} }
class B { public B(A a) {} }
```

💥 Runtime crash

---

## ✅ Solution 1: Lazy Injection

```csharp
class A
{
    private readonly Lazy<B> _b;

    public A(Lazy<B> b)
    {
        _b = b;
    }
}
```

---

## ✅ Solution 2: Refactor (BEST)

👉 Extract shared logic into third service:

```
A → C ← B
```

---

## 🎯 Interview Insight

👉 Best answer:

> “Circular dependency is a design smell. I prefer refactoring over Lazy unless unavoidable.”

# 🔥 3. How do you implement Conditional Dependency Injection?

## ❓ Scenario

Inject service based on environment:

* Dev → FakeService
* Prod → RealService

---

## ✅ Solution

```csharp
if (builder.Environment.IsDevelopment())
{
    services.AddScoped<IService, FakeService>();
}
else
{
    services.AddScoped<IService, RealService>();
}
```

---

## 🔥 Advanced Version (Runtime Condition)

```csharp
services.AddScoped<IService>(provider =>
{
    var config = provider.GetRequiredService<IConfiguration>();

    var type = config["ServiceType"];

    return type switch
    {
        "email" => new EmailService(),
        "sms" => new SmsService(),
        _ => throw new Exception()
    };
});
```

---

👉 This is **very commonly asked**

# 🔥 4. How do you decorate a service (Decorator Pattern with DI)?

## ❓ Scenario

Add logging to service without modifying original code.

---

## ✅ Solution

```csharp
public class LoggingService : IMessageService
{
    private readonly IMessageService _inner;

    public LoggingService(IMessageService inner)
    {
        _inner = inner;
    }

    public void Send(string msg)
    {
        Console.WriteLine("Logging...");
        _inner.Send(msg);
    }
}
```

---

### Registration

```csharp
services.AddScoped<IMessageService, EmailService>();
services.Decorate<IMessageService, LoggingService>();
```

👉 (Requires Scrutor package)

---

## 🎯 Interview Insight

👉 Shows:

* Open/Closed principle
* Real enterprise pattern

# 🔥 5. How do you manage DbContext lifetime correctly in large apps?

## ❓ Problem

DbContext is Scoped — but used in:

* Controllers
* Background jobs
* Services

---

## ✅ Best Practice

* Always Scoped
* Never Singleton
* Use factory for background jobs

```csharp
services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(conn));
```

---

## 🎯 Advanced Answer

👉 Mention:

* Unit of Work pattern
* Transactions

# 🔥 6. How would you implement Plugin Architecture using DI?

## ❓ Scenario

System should load new modules without code change.

---

## ✅ Approach

```csharp
var assemblies = Directory.GetFiles("plugins", "*.dll");

foreach (var file in assemblies)
{
    var assembly = Assembly.LoadFrom(file);

    var types = assembly.GetTypes()
        .Where(t => typeof(IService).IsAssignableFrom(t));

    foreach (var type in types)
    {
        services.AddScoped(typeof(IService), type);
    }
}
```

---

## 🎯 Interview Insight

👉 This is **architect-level answer**

# 🔥 7. How do you prevent memory leaks in DI?

## ❓ Problem

* Singleton holding large objects
* Not disposing Scoped services

---

## ✅ Solution

* Use `IDisposable`
* Avoid static references
* Use correct lifetime

```csharp
public class MyService : IDisposable
{
    public void Dispose()
    {
        // cleanup
    }
}
```

---

👉 Shows production awareness
